# Примеры использования платформы

## Пример 1: Физические теории (ваш кейс)

### Структура домена "Physics Theories"

#### Типы узлов:
1. **Axiom** (Аксиома)
   - Базовые постулаты
   - Цвет: #FF6B6B
   - Поля: `statement` (текст постулата), `author`, `year`

2. **Theory** (Теория)
   - Теоретические построения
   - Цвет: #4ECDC4
   - Поля: `equations` (формулы), `domain` (область применения)

3. **Experiment** (Эксперимент)
   - Экспериментальные данные
   - Цвет: #45B7D1
   - Поля: `year`, `accuracy`, `methodology`, `results`

4. **Interpretation** (Трактовка)
   - Интерпретация результатов
   - Цвет: #96CEB4
   - Поля: `interpretation_type`, `confidence_level`

5. **Concept** (Концепция/Картина мира)
   - Верхний уровень - целостные теории
   - Цвет: #FFEAA7
   - Поля: `completeness_score`, `paradigm`

#### Типы связей:
1. **derives_from** (следует из)
   - Теория → Аксиома
   - Вес: +0.8

2. **supports** (подтверждает)
   - Эксперимент → Теория
   - Вес: +1.0

3. **contradicts** (противоречит)
   - Эксперимент → Теория
   - Вес: -1.0

4. **interprets** (интерпретирует)
   - Трактовка → Эксперимент
   - Вес: +0.5

5. **part_of** (является частью)
   - Теория → Концепция
   - Вес: +0.6

### Пример графа:

```
Концепция: "Эфиродинамика"
  ├─ Теория: "Механическая теория эфира"
  │   ├─ Аксиома: "Существование эфира"
  │   └─ Эксперимент: "Опыт Майкельсона-Морли"
  │       └─ Трактовка: "Эфир неподвижен" [contradicts]
  │       └─ Трактовка: "Сокращение Лоренца" [supports]
  └─ Теория: "Вихревая модель атома"
      └─ Эксперимент: "Спектральные линии"
          └─ Трактовка: "Вихревые структуры" [supports]
```

### Расчет рейтинга концепции:

```javascript
Consistency = (supports - contradicts) / total_edges
Coherence = connected_nodes / total_nodes
Connectivity = edges_count / (nodes_count * (nodes_count - 1))
Overall = 0.4 * Consistency + 0.3 * Coherence + 0.3 * Connectivity
```

## Пример 2: Математические доказательства

### Структура домена "Mathematics"

#### Типы узлов:
1. **Axiom** (Аксиома)
2. **Definition** (Определение)
3. **Theorem** (Теорема)

5. **Corollary** (Следствие)
6. **Conjecture** (Гипотеза)

#### Типы связей:
1. **proves** (доказывает)

3. **generalizes** (обобщает)
4. **specializes** (является частным случаем)

### Пример графа:

```
Аксиома: "Аксиомы Пеано"
  └─ Определение: "Натуральные числа"
      └─ Теорема: "Принцип математической индукции"
          ├─ Лемма: "База индукции"
          └─ Лемма: "Индукционный переход"
              └─ Теорема: "Формула суммы арифметической прогрессии"
                  └─ Следствие: "Сумма первых n натуральных"
```

## Пример 3: Образовательный трек "Web Development"

### Структура домена "Web Development Skills"

#### Типы узлов:
1. **Skill** (Навык)
   - Поля: `difficulty` (1-5), `time_to_learn` (hours)

2. **Topic** (Тема)
3. **Project** (Проект)
4. **Resource** (Ресурс - книга, курс)

#### Типы связей:
1. **requires** (требует)
2. **recommended_before** (рекомендуется перед)
3. **practices** (практикуется в)
4. **teaches** (обучает)

### Пример графа:

```
Навык: "HTML Basics" [difficulty: 1, time: 10h]
  └─ Навык: "CSS Basics" [difficulty: 2, time: 20h]
      ├─ Навык: "CSS Flexbox" [difficulty: 3, time: 10h]
      └─ Навык: "CSS Grid" [difficulty: 3, time: 10h]
          └─ Навык: "Responsive Design" [difficulty: 3, time: 15h]
              └─ Проект: "Portfolio Website"
                  └─ Ресурс: "MDN Web Docs"
```

### Прогресс пользователя:

```javascript
// Пользователь отмечает навыки как "освоенные"
user.completedSkills = ["HTML Basics", "CSS Basics"]

// Система рассчитывает:
// 1. Какие навыки доступны для изучения (все prerequisites выполнены)
// 2. Рекомендуемый порядок изучения
// 3. Прогресс в целом (30 hours / 100 total hours = 30%)
```

## Общие паттерны использования

### 1. Иерархические структуры
Используйте связь `part_of` для создания дерева:
```
Концепция
  └─ Теория 1
  └─ Теория 2
      └─ Подтеория 2.1
      └─ Подтеория 2.2
```

### 2. Зависимости
Используйте связь `depends_on` или `requires`:
```
Навык А → требует → Навык Б → требует → Навык В
```

### 3. Противоречия
Используйте связь `contradicts` для выявления конфликтов:
```
Эксперимент 1 → contradicts → Теория А
Эксперимент 2 → supports → Теория А
```
Рейтинг Теории А будет ниже из-за противоречия.

### 4. Эволюция знаний
Версионируйте узлы с временными метками:
```
Теория v1.0 (1905)
  └─ Теория v2.0 (1915) [улучшенная версия]
      └─ Теория v3.0 (2020) [современная версия]
```

### 5. Мультиязычность
Используйте поле `translations`:
```json
{
  "title": "Theory of Relativity",
  "translations": {
    "ru": {
      "title": "Теория относительности",
      "content": "..."
    },
    "de": {
      "title": "Relativitätstheorie",
      "content": "..."
    }
  }
}
```

## Советы по организации знаний

1. **Начните с верхнего уровня**
   - Определите основные концепции
   - Разбейте на подтеории
   - Добавьте детали

2. **Используйте тэги**
   - Группируйте похожие узлы
   - Облегчите поиск

3. **Добавляйте метаданные**
   - Даты, авторы, ссылки
   - Уровень доверия
   - Источники

4. **Поддерживайте консистентность**
   - Используйте одинаковые типы узлов для похожих сущностей
   - Следуйте naming conventions
   - Документируйте свою структуру

5. **Регулярно проверяйте рейтинги**
   - Находите противоречия
   - Заполняйте пробелы
   - Укрепляйте слабые места
