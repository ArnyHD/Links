# Knowledge Graph Platform - Обзор проекта

## Что это?

Knowledge Graph Platform - универсальная платформа для организации и анализа знаний в виде связанных графов. Изначально создана для работы с альтернативными физическими теориями и эфиродинамикой, но спроектирована так, чтобы поддерживать любые области знаний.

## Ключевая идея

**Представление знаний как граф:**
- **Узлы** - элементы знаний (теории, эксперименты, аксиомы, навыки, etc.)
- **Связи** - отношения между элементами (поддерживает, противоречит, следует из, etc.)
- **Рейтинги** - автоматический расчет качества концепций на основе структуры графа

## Основные возможности

### 1. Гибкая структура данных
Пользователи сами определяют:
- Какие типы узлов существуют в их домене
- Какие поля должны быть у каждого типа узла
- Какие типы связей возможны
- Какой смысл несет каждая связь (поддержка, противоречие, зависимость)

**Пример для физики:**
```
Типы узлов: Аксиома, Теория, Эксперимент, Трактовка, Концепция
Типы связей: следует из, подтверждает, противоречит, интерпретирует
```

**Пример для обучения:**
```
Типы узлов: Навык, Тема, Проект, Ресурс
Типы связей: требует, рекомендуется перед, практикуется в, обучает
```

### 2. Автоматический анализ

Платформа автоматически рассчитывает:

- **Непротиворечивость (Consistency)**
  - Сколько связей поддерживают концепцию vs противоречат
  - Выявляет слабые места и конфликты

- **Целостность (Coherence)**
  - Насколько полна структура
  - Какие части недостаточно развиты

- **Связность (Connectivity)**
  - Насколько хорошо элементы связаны между собой
  - Изолированные части графа

- **Общий рейтинг (Overall)**
  - Взвешенная комбинация всех метрик
  - Сравнение разных концепций

### 3. Интерактивная визуализация

- Граф знаний в виде интерактивной диаграммы
- Разные алгоритмы расположения (force-directed, иерархический, круговой)
- Фильтрация по типам узлов и связей
- Поиск путей между узлами
- Zoom, pan, drag

### 4. Многоязычность

- Интерфейс на русском и английском
- Контент узлов можно переводить
- Автоопределение языка пользователя

### 5. Collaborative работа

- Несколько пользователей могут работать над одним графом
- Комментарии к узлам и связям
- История изменений
- Модерация контента

## Основные сущности

### Domain (Домен)
- Область знаний (физика, математика, обучение, etc.)
- Содержит свои типы узлов и связей
- Может быть публичным или приватным

### Node Type (Тип узла)
- Определяет категорию узлов
- Имеет JSON схему для валидации данных
- Примеры: Теория, Эксперимент, Навык, Документ

### Node (Узел)
- Конкретный элемент знаний
- Содержит заголовок, контент, кастомные данные
- Может иметь переводы
- Имеет статус (черновик, опубликован, архивирован)

### Edge Type (Тип связи)
- Определяет семантику связи
- Имеет вес для расчета рейтингов
- Может быть направленной или ненаправленной
- Примеры: поддерживает (+1), противоречит (-1), следует из (+0.5)

### Edge (Связь)
- Соединяет два узла
- Имеет тип и направление
- Может содержать дополнительные данные

### Rating (Рейтинг)
- Автоматически рассчитывается для узлов
- Разные типы метрик
- Сохраняет историю и детали расчета

## Преимущества перед альтернативами

### vs Notion/Confluence
- Специализирован для связанных знаний
- Автоматический анализ структуры
- Визуализация графа
- Гибкая типизация

### vs Neo4j/OrientDB
- User-friendly интерфейс
- Готовая система рейтингов
- Многоязычность из коробки
- Не требует знания Cypher/Gremlin

### vs Mind Maps
- Поддержка сложных связей (не только дерево)
- Разные типы узлов и связей
- Автоматические метрики
- Collaborative работа

### vs Wikipedia
- Структурированные связи
- Количественная оценка качества
- Поддержка альтернативных точек зрения
- Гибкая организация контента

## Технические особенности

### Граф в реляционной БД
- PostgreSQL с JSONB для гибкости
- Рекурсивные запросы для обхода графа
- При необходимости можно добавить Neo4j

### Масштабируемость
- Stateless backend (легко горизонтально масштабировать)
- Кэширование на разных уровнях
- Code splitting на frontend
- Lazy loading данных

### Безопасность
- JWT authentication
- Role-based access control
- Input validation
- XSS/CSRF protection

## Начало работы

```bash
# Клонировать репозиторий
git clone <repo-url>
cd knowledge-graph-platform

# Установить зависимости
npm install

# Запустить базу данных
docker-compose up -d postgres

# Настроить переменные окружения
cp apps/backend/.env.example apps/backend/.env

# Запустить миграции
npm run migration:run --workspace=@kgp/backend

# Запустить приложение
npm run dev
```
